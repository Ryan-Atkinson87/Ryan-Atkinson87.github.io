Trive AquaSense Documentation

Explains how to run the project on Raspberry Pi as a persistent service using systemd. Includes logging, service recovery, directory expectations and versioning.

Deployment

This guide explains how to deploy Trive AquaSense as a long running service on a Raspberry Pi using systemd.

Install the Systemd Service

sudo cp mqtt_aquarium_monitoring_example.service /etc/systemd/system/mqtt_aquarium_monitoring.service
sudo systemctl enable mqtt_aquarium_monitoring
sudo systemctl start mqtt_aquarium_monitoring

Service Behaviour

  • Restarts automatically on failure
  • Runs through your virtual environment
  • Starts on boot

Logging

  • Configured in logging_setup.py
  • RotatingFileHandler at /var/log/monitoring_service.log

Configuration Files

  • .env for secrets
  • config.json for sensors and settings

Versioning Rules

MAJOR.MINOR.PATCH

Hardware Tests

pytest -m hardware
