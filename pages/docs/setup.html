Setup and Installation

Follow this guide to install Trive AquaSense on a Raspberry Pi and prepare the environment for data collection and transmission to ThingsBoard.

To find setup instructions for ThingsBoard community edition, please click here.

1. Requirements

  • Raspberry Pi running Raspberry Pi OS
  • Python 3.11 or later
  • Virtual environment recommended
  • ThingsBoard instance and device access token

2. Clone the Repository

git clone https://github.com/Ryan-Atkinson87/mqtt_aquarium_monitoring
cd mqtt_aquarium_monitoring

3. Create a Virtual Environment

python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

4. Configure Environment Variables

Edit your .env file. Required keys:

ACCESS_TOKEN=your_thingsboard_token
THINGSBOARD_SERVER=your_tb_host
LOG_LEVEL=INFO

5. Configure Sensors in config.json

The file defines polling intervals, device name, and sensor list. Example:

{
  "device_name": "AquariumPi",
  "poll_period": 2,
  "sensors": [
    {
      "id": "water_temp",
      "type": "DS18B20",
      "pin": 4,
      "interval": 5
    }
  ]
}

For a full working example please see config.example.json within the github repository.

6. Run the Agent

python -m monitoring_service.main

7. Enable systemd Service

For a full working service file example please see mqtt_aquarium_monitoring_example.service within the github repository.

sudo cp mqtt_aquarium_monitoring_example.service /etc/systemd/system/mqtt_aquarium_monitoring.service
sudo systemctl enable mqtt_aquarium_monitoring
sudo systemctl start mqtt_aquarium_monitoring
