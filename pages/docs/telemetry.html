Explains the telemetry scheduler, read intervals, retries, backoff logic and ThingsBoard formatting for outbound messages.

Telemetry

The telemetry pipeline manages timed reads, error retries, backoff logic and forwarding of telemetry to ThingsBoard.

Telemetry Responsibilities

  • Enforce per sensor intervals
  • Retry read failures
  • Apply exponential backoff
  • Publish via MQTT or HTTP

Payload Example

{
  "ts": 1730022000000,
  "values": {
    "cpu_usage": 23.7,
    "cpu_temp_c": 51.2
  }
}

Attributes Sent on Startup

  • hostname
  • mac_address
  • ip_address
  • device_name

Error Handling

Domain specific exceptions with no bare except blocks.
