Trive AquaSense Documentation

High level explanation of Trive AquaSense, its purpose, capabilities and design goals. Includes architectural philosophy and feature summary.

Overview

Trive AquaSense is a modular Raspberry Pi based monitoring service designed to collect telemetry from environmental sensors and push the data to ThingsBoard. The system prioritises modularity, reliability, easy configuration and production grade behaviour.

Core Features

  • Pluggable sensor drivers with a consistent interface
  • Config driven sensor loading using config.json
  • Robust telemetry collector with retries and per sensor intervals
  • MQTT or HTTP transport via a ThingsBoard client wrapper
  • Rotating file logs and structured error handling
  • Systemd service for resilient deployment on Raspberry Pi

Project Goals

  • Support new sensors without modifying core logic
  • Provide a clean module layout for long term maintenance
  • Handle network problems through safe reconnect logic
  • Ensure telemetry accuracy through backoff and retry rules
  • Expose canonical telemetry keys for all sensors
