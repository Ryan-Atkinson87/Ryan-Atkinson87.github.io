Architecture

The architecture is modular and splits responsibilities across well defined packages. This page explains how the internal components cooperate.

High Level Diagram

config_loader   -> loads .env and config.json
attributes      -> gathers device metadata
sensors.factory -> builds sensor objects
telemetry       -> schedules reads and publishes data
TBClientWrapper -> MQTT client for ThingsBoard
logging_setup   -> central logging config
agent.py        -> orchestrates startup and loop

Sensor Layer

  • Each sensor extends BaseSensor in sensors/base.py
  • Implements read() returning canonical telemetry dict
  • Optional GPIO or I2C validation via _check_pin()

Factory Layer

Located in sensors/factory.py:

  • Validates sensor IDs and types
  • Instantiates correct class based on config
  • Ensures safe defaults and explicit errors

Telemetry Collector

  • Manages per sensor intervals
  • Retries failed reads with exponential backoff
  • Formats ThingsBoard telemetry payloads
  • Hands off transport to client wrapper

Client Layer

TBClientWrapper.py:

  • MQTT transport
  • Auto reconnect handling
  • Sends device attributes on connect

Logging

  • RotatingFileHandler
  • Configurable log level
  • Timestamps and module labels
